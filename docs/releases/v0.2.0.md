# v0.2.0 Release Notes (Draft)

发布日期：2026-04-12  
状态：Draft

## 亮点

1. 列表能力升级
- containers/images/volumes/networks 全部支持服务端分页。
- containers/images 支持服务端筛选。

2. 审计能力升级
- 审计接口支持分页与筛选。
- 审计页支持防抖查询和取消过期请求。

3. 可分享筛选链接
- containers/images/audit 页面筛选状态可写入 URL。
- 刷新与分享链接可保留筛选条件。

## 迁移与兼容

- ✅ 不需要后端接口重命名迁移。原有接口路径不变。
- ✅ 列表接口响应体仍是数组，旧客户端可继续使用。
- ✅ 新增 query 参数均为可选，不传即保持旧行为。

## 建议联调检查清单

- [ ] `GET /api/containers?from=0&limit=20&q=nginx&status=running`
- [ ] `GET /api/images?from=0&limit=20&q=alpine`
- [ ] `GET /api/audit?from=0&limit=200&q=create&result=failed`
- [ ] 前端切换容器/镜像/审计页时，URL 是否携带筛选参数。
- [ ] 刷新页面后筛选状态是否保持。

## 已知限制

- 当前 volumes/networks 仅支持分页，尚未开放服务端关键词筛选。
- 列表分页元信息通过响应头返回，暂未统一成 JSON 包装结构。
