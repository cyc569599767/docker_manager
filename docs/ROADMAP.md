# Roadmap

## 近期（P1）

1. 文案与国际化收敛
- 全面统一前端中文文案（移除临时英文文案）。
- 统一错误提示风格（用户可读 + 调试可追踪）。

2. 日志能力增强
- 容器日志支持增量游标（`from`）与自动刷新策略。
- 镜像拉取进度可选 SSE/WebSocket，替代高频轮询。

3. API 规范化
- 统一响应字段命名（camelCase）。
- 增加错误码（`code`）与请求追踪字段（`requestId`）。

4. 测试补齐
- 后端：分页/筛选/校验逻辑单测。
- 前端：关键交互（筛选、分页、URL 同步）组件测试。

## 中期（P2）

1. 审计日志治理
- 按大小或日期滚动归档。
- 提供审计清理策略与保留周期配置。

2. 权限与安全
- 引入轻量鉴权（本地 token / basic auth）。
- 收紧 CORS 策略、补充敏感操作防护。

3. 性能可观测性
- 增加慢请求日志与关键接口耗时指标。
- 前端增加关键页面渲染耗时埋点。

## 长期（P3）

1. 多节点 / 远程 Docker 管理
- 支持多个 Docker endpoint 配置与切换。
- 支持只读模式与多角色权限。

2. 任务系统抽象
- 长任务统一队列与状态持久化（拉取、导入、批量操作）。
